# docker 容器管理

容器有以下内容：

# 容器标识符

docker ps 列出的信息第一列是容器的ID，也可以--name起一个别名

# 查询容器信息

docker inspect

# 查询容器日志

docker logs

# 进入容器

docker exec 容器名 命令

docker exec -it 容器名 命令 以root身份进入，以交互模式执行指令

# 多容器管理

理念：一个容器一个进程，如果一个服务由多个进程组成，需要创建多个容器组成一个系统。

统一cgi一个容器、spp服务一个容器，ckv一个容器。

容器编排工具：docker compose，允许用户在一个yaml文件中定义一组相关联的应用容器，根据定义的--link参数，对启动的优先级进行自动排序，执行docker-compose up就可以把同一个服务中多个容器依次创建和启动，执行docker-compose stop就可以把停止这个服务。

# docker镜像管理

docker images -a 查看本地所有安装的镜像

docker history 查看镜像分了多少层
