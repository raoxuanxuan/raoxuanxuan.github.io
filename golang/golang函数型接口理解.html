# golang函数型接口理解

在go中，当需要传入一个接口类型的形参时，当且仅当某种类型所有实现了这个接口的所有方法时，就可以作为实参传递给这个形参。举个例子

func process(contents []string, handler Handler) { ​ for _, v := range contents { ​ handler.Do(v) ​ } }

上面这个函数会调用hander接口的do方法来处理切片里每一个string。

接口Handler的定义如下：

Handler interface { ​ func Do(str string) string }

如果我们想对每个字符串加一个前缀，我们需要先定义一个结构体来实现这个Handler接口。

type prefixString string

func (p prefixString) Do(str string) string { return "prefix" + str }

然后调用process函数，传入类型为prefiexString的参数即可。

这种方式有一个不方便的地方在于：

即使我们有一个原型满足接口里方法的函数addPrefix:

func addPrefix(str string) string { return "prefix" + str }

也必须再定义一个类型prefixString，让这个类型去实现接口里的Do方法，只不过Do方法就是addPrefix的实现，无法将addPrefix直接传给handler参数。

addPrefix和Do的传参和返回类型都是一样的。

函数型接口其实就是一个转换，把原型满足接口里的方法的函数转化为接口类型。这个函数型接口实现Do方法，实现的代码中，把Do的参数传给自己。

type FunctionHandler func(str string) string

func (f FunctionHandler) Do(str string) string { return f(str)}

然后调用的时候，就不用额外再定义类型去实现Do方法。

process(contents, FunctionHandler(addPrefix))
